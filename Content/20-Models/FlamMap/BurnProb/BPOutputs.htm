After running Burn Probability, note that:

  • Outputs are relative within each landscape and CANNOT be compared across landscapes.
  • Conditional Flame Length is an output of the burn probability model useful for hazard and risk calculations. It represents a weighted average of the flame length in each pixel over multiple simulations rather than an absolute output from one simulation. It is therefore nearly always lower than a single model run flame length output using the same input values.

Landscape Burn Probability outputs may be analysed within IFTDSS, or exported for use in other applications. Specific information for each option is presented in its own Help Center Topic:

  • LBP Outputs within IFTDSS (Map and Report)
  • LBP Outputs for export outside of IFTDSS

Both output approaches will provide burn probability and conditional flame length information.

Burn Probability

Burn probabilities represent the likelihood of a given location on your landscape burning. Burn probabilities are related to the sizes of fires that occur on a given landscape. Large fires produce higher probabilities than small fires. Since fire size is a function of both the rate of spread, and the duration of a fire, treatments or weather conditions that reduce the spread rate will lower the burn probability.

Within Landscape Burn Probability, randomly located ignition points are used to simulate fires. The number of ignitions is determined by IFTDSS to most efficiently produce outputs for the user. See the Ignitions topic for details.

Burn Probability is the fraction of the number of fires that burned each burnable pixel (0.0 to 1.0). It is calculated as:

Burn Probability = # times burned / total number of ignitions

Burn probability value of 0 indicates the pixel was never burned and a value of 1 indicates it was always burned.

A value of 1 indicates that the pixel burned for all fires simulated. Where 0 indicates that fire did not burn the pixel. This can be because a fire never reached the pixel, or a fire reached the pixel or started within the pixel but it was unable to burn out of the pixel because the fire spread rate was too slow.

Conditional Flame Length

Conditional Flame length represents the flame length (fire intensity) of each pixel, as a composite of all the simulated fires burned on any given pixel. It relies on the proportion of all simulated MTT run outputs and the midpoint flame length. For example, if flame lengths on a pixel were 4-6 feet in 92% of the simulations, 2-4 feet in 6% of simulation, and 6-8 2% of the time, its conditional flame length would be calculated as follows:

Flame Length Mid Point FL   Proportion of Flame Length   Mid point * proportion
0-2 ft 1 x 0 = 0
2-4 ft 3 x 0.06 = 0.12
4-6 ft 5 x 0.92 = 4.6
6-8 ft 7 x 0.02 = 0.14
8-12 ft 10 x 0 = 0
>12 ft 25 x 0 = 0
