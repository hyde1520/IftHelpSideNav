User List

User List is where all user administration occurs in with the exception of FTEM users (See the FTEM Administration topic for those details).

The left panel of User List contains filters to help you narrow the list of users based on user attributes (active vs disabled, government or non-goverment etc.). There is also a field in the top of the left panel that allows you to search for a specific user name or email.

The center of the User List contains the list of users. Clicking on a user will display a list of administration options for that user's account.

The right panel of User List contains User metadata.

The right panel will remain open as long as a user is selected and user options are displayed. This allows admins to mouse over and copy any attribute from the metadata field for bug reporting, or do assist users (for example, providing a copy of the users temporary password).

Filtering and User Definitions

User attributes which you may use to filter users in the left panel include:

  • Account Status

    • Pending: Accounts which have been requested but not yet accessed. These are either accounts from government users who have not yet used their initial access code, or account from non-government users who have not yet been approved by an administrator.

    • User: The main role in . Users may access and perform any modeling, reporting, or downloading function.

    • Administrator: This role has all the same privileges as a "User" with the added capacity to manage users or perform system administration.

    • Disabled: Accounts which have been disabled due to inactivity, or having their account manually disabled by an administrator.

  • Agency

    • Government: A user affiliated with an email belonging to one of federal land management agencies (Bureau of Indian Affairs, Bureau of Land Management, Fish and Wildlife Service, National Park Service, USDA Forest Service) or Firenet.

    • Non-government: All other users not included in the above list.

User Options

This section provides a brief overview of buttons that administrators will see when they select a user in the User List. For more detail on each of these options, see the User Administration topic.

When an administrator selects a user account by clicking on it, they will see a drop-down with a list of options for demonstration purposes. These include:

  • Grant User Privileges: Grants user privileges to disabled accounts or pending accounts.

  • Grant Administrator Privileges: Grants administrator privileges.

  • Disable Privileges: Disables the user's privileges, user will no longer be able to gain access to their account.

  • Archive: Disables the user's privileges, user will no longer be able to gain access to their account.

  • Reset Password: Assigns a temporary password to a user and automatically emails that temporary password to the user's email.

  • Unlock: Unlocks a users account and automatically emails that temporary password to the user's email.

  • Allow Password Change: Removes the 24 hour lock on resetting passwords for accounts that have been reset more than once in a 24-hour period.

  • Get Audit Log: Opens a log of account activity for the users account.
