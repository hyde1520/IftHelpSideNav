User Management Cheat sheet

Granting or Restoring Access

Depending on the user's situation, there are different buttons that are visible to admins who need to assist a user in regaining access to their account. Each is summarized below and includes:

  • Grant User Privileges

  • Reset Password

  • Unlock

Grant User Privileges

"Grant User Privileges" appears on accounts for users who need either need first-time access to , or to regain access because their account has lapsed after 90 days of inactivity.

Grant User Privileges makes the account active, and automatically sends the user a temporary password they may use to gain access to their account.

Reset Password

"Rest Password" is always visible. It allows and admin to reset a users password.

Reset Password automatically sends the user a temporary password they may use to gain access to their account.

Reset Password will not unlock a locked account. Locked accounts must be unlocked using the Unlock button, described in the next sections.

Users can accomplish the same task as and administrator using "Reset Password" if they follow the "Forgot Password" prompt in the log in screen. Though they may be contacting an administrator to reset their password if they've forgotten their security question and cannot complete the "Forgot Password" process.

Unlock

"Unlock" appears on accounts for users who have locked their account by entering the incorrect password too many times. Accounts are locked if there are too many unsuccessful login attempts within a 24-hour period.

Unlock will unlock the account and automatically send the user an email containing a new temporary password they can use to regain access to their account.

User Accounts lock after the fourth unsuccessful login attempt in 24 hours. The user interface makes reference to "three" attempts, so some users may contact an administrator saying their account is locked, even though the administrator can see in the User List that the account is not yet locked.

If a user requests an unlock or password reset more than once in a 24-period period, they may be prevented from completing the process, and will see a messaging on the password reset page that informs them "Exception: Your password was updated recently and cannot be modified until 24 hours since your previous change".

Removing Access and Archiving

Both of the below options are always available:

Disable Privileges

"Disable Privileges" disables an account and sends the user an email notification that their account has been disabled.

To re-enable an account which has been disabled, use the "Grant User Privileges" button.

Archive

"Archive" will remove an account from the User List, users will be unable to login to those accounts. Users will not receive an email notification. This option is intended to remove dis-used accounts. For example, a user may create a second account and no longer need their old account, in which case an admin can archive the older account.
