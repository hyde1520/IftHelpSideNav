About User Administration

There are two roles in :

  • Users: Those with accounts who can access the system, generate landscapes, run models, and perform other tasks.

  • Administrators. Administrators can assist users in creating and maintaining their accounts. This section addresses the key steps in managing user accounts.

All user management activities for are conducted from the User List, visible under user options only for administrators.

Fuels Treatment Effectiveness Monitoring (FTEM) users have a separate set of permissions just for FTEM. Managing FTEM is discussed in the Administrators FTEM Topics within the FTEM section of Help Center.

Throughout this topic you'll see references to scenarios in which users contact administrators for support. That contact is carried out through the Team's Freshdesk support ticket system. New administrators should contact a member of the Development Team if details about Freshdesk are required.

User Access and Passwords

There are three variations of approving user access or assisting users in gaining access to their accounts. Each of these are described in further depth below:

  • Grant User Privileges

  • Reset Password

  • Unlock

Grant User Privileges

The Grant User Privileges button appears on accounts for users who need either need first-time access to , or existing users who have had their accounts disabled due to inactivity.

Grant User Privileges makes the account active, and automatically sends the user a temporary password they'll use to gain access to their account.

Example: Grant User Privileges for a New User

  1. A non-federal user registers with a new account. An administrator reviews the request and decides to approve the account.

  2. The administrator navigates to the User List and searches to find the user's name.

  3. The administrator single-clicks the user name to view options and clicks Grant User Privileges.

  4. automatically sends an email to that user containing a temporary password.

  5. The user logs into with the temporary password and updates their password.

  6. The user may not access normally.

 Example: Grant User Privileges for a Disabled User

  1. A user attempts to log in with a disabled account and sees a system-generated message to contact an administrator. The user contacts administrator to re-enable their account.

  2. The administrator navigates to the User List and searches to find the user's name.

  3. The administrator single-clicks the user name to view options and clicks Grant User Privileges.

  4. automatically sends an email to that user containing a temporary password.

  5. The user logs in with the temporary password and updates their password.

  6. The user may now log into .

Reset Password

The Reset Password button is always visible. It allows an administrator to reset a users password.

Reset Password automatically sends the user a temporary password they may use to gain access to their account.

Reset Password will not unlock a locked account. Locked accounts must be unlocked using the Unlock button, described in the next sections.

Users can accomplish the same task as an administrator using "Reset Password" if they follow the "Forgot Password" prompt in the log in screen. Though they may be contacting an administrator to reset their password if they've forgotten their security question and cannot complete the "Forgot Password" process.

Example: Resetting a Password

This will send a new password, it will not unlock an account if the account is locked.

  1. User contacts an administrator because they are unable to reset their password. This case can occur if the user has forgotten their security question and cannot complete the "forgot password" prompt themselves.

  2. An administrator navigates to the User List and searches to find the user's name.

  3. An administrator single-clicks the user name to view options and clicks Reset Password.

  4. automatically sends an email to that user containing a temporary password.

  5. The user logs in with the temporary password and updates their password.

  6. The User may now access their account using their newly updated password.

Unlock

The Unlock button appears on accounts for users who have locked their account by entering the incorrect password too many times. Accounts are locked if there are too many unsuccessful login attempts within a 24-hour period.

Unlock will unlock the account and automatically send the user an email containing a new temporary password they can use to regain access to their account.

User Accounts lock after the fourth unsuccessful login attempt in 24 hours. The user interface makes reference to "three" attempts, so some users may contact an administrator saying their account is locked, even though the administrator can see in the User List that the account is not yet locked.

If a user requests an unlock or password reset more than once in a 24-period period, they may be prevented from completing the process, and will see a messaging on the password reset page that informs them "Exception: Your password was updated recently and cannot be modified until 24 hours since your previous change." Administrators may assist the user cases such as this by using the Allow Password Change button.

Example: Account Unlock

  1. A user receives a system-generated message that their account is locked. This can occur if a user attempts to login with an invalid password multiple times in a 24 hour period.

  2. An administrator navigates to the User List and searches to find the user's name.

  3. An administrator single-clicks the user name to view options and clicks Unlock.

  4. automatically sends an email to that user containing a temporary password.

  5. The user logs in with the temporary password and updates their password.

  6. The user may not access their account normally.

Allow Password Change

The Allow Password Change button allows a user to change their password an additional time within a 24-hour period. By default, users may only change their password once in a 24-period. This security feature helps prevent unauthorized account access. However, there are instances where a user may need to change their password more than once in a 24-hour period, such as when a user resets their password but forgets or mis-types the new password. At that time the user may contact an administrator who may use the Allow Password Change button to help the user change their password and regain access.

This button "resets the clock" on a user account, allowing a user to update their password one additional time if needed

This button only appears as an option on accounts that have had the password updated within 24 hours.

Example: Allowing another password reset

  1. A user contacts an demonstrator because they are unable to reset their password. This case can occur if the user forgotten their security question and cannot complete the "forgot password" prompt themselves.

  2. An administrator navigates to the User List and searches to find the user's name.

  3. An administrator single-clicks the user name to view options and clicks Reset Password.

  4. automatically sends an email to that user containing a temporary password.

  5. The user logs in with the temporary password and updates their password.

  6. The user attempts to log in again and realizes their new password is not working.

  7. From this point, if a user tries the "forgot password" prompt and attempts to reset their password, they will receive a notice preventing them from completing the process. It is at this point they would reach out to an administrator for assistance.

  8. An administrator will locate the user in the User List, click the users name to show account options and click Allow Password Change. This will remove the 24-hour block on the user account. No email is sent to the user. From this point, the administrator may choose to instruct the user to re-attempt the "forgot password" prompt or choose to reset the password using the options in the User List.

    If a user is still on the password reset interface after the administrator clicks the Allow Password Change, they will be able to reset their password by with the same temporary password so long as they have not attempted to use it more than three times.

Removing Access and Archiving

Both of the below options are always available:

Disable Privileges

The Disable Privileges button disables an account and sends the user an email notification that their account has been disabled.

To re-enable an account which has been disabled, use the Grant User Privileges button.

When an account is disabled, it is possible for the email associated with that account to be used in the creation of a new account.

Example: Disable Privileges

  1. A user requests their account be disabled or an administrator determines an account should be disabled.

  2. An administrator navigates to the User List and searches to find the user's name.

  3. An administrator single-clicks the user name to view options and clicks Disable Privileges.

  4. automatically sends an email to that user informing them their account has been disabled.

Archive

The Archive button will disable an account and the user users will not receive an email notification. This option is intended for instances where an account needs to be disabled but the user does not need to be notified. Such cases could include instances where a user with multiple accounts is already in communication with the administrator and asks for the unneeded account to be removed.

Example: Disable Privileges

  1. A user requests or administrator determines an account should be archived.

  2. An administrator navigates to the User List and searches to find the user's name.

  3. An administrator single-clicks the user name to view options and clicks Archive.

  4. The account is disabled and no email notification is sent.
